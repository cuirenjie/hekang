# 首页预约发邮件说明

这份文档是给以后维护 `woohongmedical.com` 首页预约表单用的。

## 最常用命令速查

### 重启服务

```bash
systemctl restart booking-api
```

### 查看服务状态

```bash
systemctl status booking-api
```

### 查看最近日志

```bash
journalctl -u booking-api -n 50 --no-pager
```

### 查看本机接口是否正常

```bash
curl http://127.0.0.1:3001/api/health
```

### 查看域名接口是否正常

```bash
curl https://woohongmedical.com/api/health
```

### 前端改完重新打包

```bash
cd /www/wwwroot/woohongmedical.com
npm run build
```

### 手工测试发邮件

```bash
curl -X POST https://woohongmedical.com/api/bookings \
  -H "Content-Type: application/json" \
  -d '{"name":"测试客户","phonePrefix":"+852","phone":"61234567","date":"2026-03-25","service":"影像科","notes":"官网预约邮件测试"}'
```

## 当前功能链路

1. 用户在首页填写预约表单
2. 前端提交到 `/api/bookings`
3. Node 服务接收数据并拼接邮件内容
4. 阿里云邮件推送把邮件发给客服邮箱

## 关键文件

### 1. 前端表单

文件：

`/www/wwwroot/woohongmedical.com/src/components/BookingForm.tsx`

作用：

- 首页预约表单界面
- 点击“提交预约申请”时发起请求到 `/api/bookings`

如果你要改：

- 表单字段
- 默认专科
- 提交文案
- 成功提示文案

就改这个文件。

### 2. 后端接口入口

文件：

`/www/wwwroot/woohongmedical.com/server/index.js`

作用：

- 提供 `/api/health`
- 提供 `/api/bookings`
- 接收预约数据
- 校验后调用发邮件逻辑

如果你要改：

- 接口路径
- 提交频率限制
- 返回报错文案

就改这个文件。

### 3. 表单字段校验

文件：

`/www/wwwroot/woohongmedical.com/server/validation.js`

作用：

- 校验姓名、电话、日期、备注等字段

如果你要改：

- 哪些字段必填
- 字段长度限制
- 日期格式校验

就改这个文件。

### 4. 邮件标题和正文拼装

文件：

`/www/wwwroot/woohongmedical.com/server/booking-email.js`

作用：

- 根据用户提交的信息生成邮件标题
- 根据用户提交的信息生成邮件正文

如果你要改：

- 邮件标题格式
- 邮件正文内容
- 时间显示格式
- 缺省值，比如“待确认日期”

就改这个文件。

### 5. 真正发邮件的逻辑

文件：

`/www/wwwroot/woohongmedical.com/server/mailer.js`

作用：

- 调用阿里云 Direct Mail SDK
- 使用环境变量读取 AK、发件地址、收件地址

如果你要改：

- 收件邮箱
- 发件人名称
- 发件账号
- 阿里云邮件配置

通常不用改这个文件本身，优先改 `.env.local`。

### 6. 环境变量模板

文件：

`/www/wwwroot/woohongmedical.com/.env.example`

作用：

- 记录环境变量应该有哪些

实际生效的是：

`/www/wwwroot/woohongmedical.com/.env.local`

## 最常见修改

### 修改接收邮件的客服邮箱

改这个文件：

`/www/wwwroot/woohongmedical.com/.env.local`

修改这一项：

```env
DM_TO_ADDRESS="service@example.com"
```

改完以后：

- 必须重启 Node 服务

命令：

```bash
systemctl restart booking-api
```

然后检查：

```bash
curl http://127.0.0.1:3001/api/health
```

### 修改发件人显示名称

改这个文件：

`/www/wwwroot/woohongmedical.com/.env.local`

修改这一项：

```env
DM_FROM_ALIAS="客服"
```

改完以后：

- 必须重启 Node 服务

命令：

```bash
systemctl restart booking-api
```

### 修改发件账号

改这个文件：

`/www/wwwroot/woohongmedical.com/.env.local`

修改这一项：

```env
DM_ACCOUNT_NAME="service@mail.woohongmedical.com"
```

注意：

- 这个必须是阿里云邮件推送里已经可用的发信地址

改完以后：

- 必须重启 Node 服务

命令：

```bash
systemctl restart booking-api
```

### 修改阿里云 AccessKey

改这个文件：

`/www/wwwroot/woohongmedical.com/.env.local`

修改这两项：

```env
ALIBABA_CLOUD_ACCESS_KEY_ID="你的新AK"
ALIBABA_CLOUD_ACCESS_KEY_SECRET="你的新SK"
```

改完以后：

- 必须重启 Node 服务

命令：

```bash
systemctl restart booking-api
```

强烈建议：

- 如果 AK 泄露，先去阿里云控制台轮换

### 修改邮件标题和正文

改这个文件：

`/www/wwwroot/woohongmedical.com/server/booking-email.js`

例如当前这里控制标题：

- `subject`

这里控制正文：

- `textBody`

改完以后：

- 必须重启 Node 服务

命令：

```bash
systemctl restart booking-api
```

### 修改首页表单字段

改这个文件：

`/www/wwwroot/woohongmedical.com/src/components/BookingForm.tsx`

如果新增字段，比如：

- 邮箱
- 性别
- 年龄

你还要同时改：

1. `src/types.ts`
2. `server/validation.js`
3. `server/booking-email.js`

因为：

- 前端要提交这个字段
- 后端要校验这个字段
- 邮件正文也要显示这个字段

改完以后：

- 需要重新打包前端
- 需要重启 Node 服务

命令：

```bash
cd /www/wwwroot/woohongmedical.com
npm run build
systemctl restart booking-api
```

## 改完之后要不要重启

### 只改 `.env.local`

要重启 Node 服务。

不用重新 `npm run build`。

命令：

```bash
systemctl restart booking-api
```

### 只改 `server/*.js`

要重启 Node 服务。

不用重新 `npm run build`。

命令：

```bash
systemctl restart booking-api
```

### 改 `src/*.tsx` 前端页面

要重新打包前端：

```bash
cd /www/wwwroot/woohongmedical.com
npm run build
```

如果前端没有改到服务端逻辑，一般不用重启 Node。

### 同时改前端和后端

都要做：

```bash
cd /www/wwwroot/woohongmedical.com
npm run build
systemctl restart booking-api
```

## Nginx 相关

站点根目录：

`/www/wwwroot/woohongmedical.com/dist`

站点配置里需要有：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:3001;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location / {
    try_files $uri $uri/ /index.html;
}
```

改完 Nginx 后执行：

```bash
nginx -t
nginx -s reload
```

## 如果发邮件失败，优先看哪里

1. 看 Node 服务是不是还在

```bash
curl http://127.0.0.1:3001/api/health
```

2. 看日志

```bash
journalctl -u booking-api -n 100 --no-pager
```

3. 看 `.env.local` 里这些值是不是正确

```env
ALIBABA_CLOUD_ACCESS_KEY_ID=
ALIBABA_CLOUD_ACCESS_KEY_SECRET=
DM_ACCOUNT_NAME=
DM_TO_ADDRESS=
DM_FROM_ALIAS=
```

4. 看域名反代是否正常

```bash
curl https://woohongmedical.com/api/health
```
